# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest `main` | ✅ |
| Older releases | ❌ |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, email **hello@axislabs.dev** with:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if you have one)

We will acknowledge receipt within 48 hours and provide a timeline for a fix. Once resolved, we'll credit you in the release notes (unless you prefer to remain anonymous).

## Scope

This policy covers the CrewCmd codebase and its default configuration. It does not cover:

- Third-party dependencies (report those to the respective maintainers)
- User-deployed instances with custom configurations
- Infrastructure or hosting providers
